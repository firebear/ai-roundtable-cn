## 🚀 重新加载步骤（重要！）

### 为什么需要重新加载？

Chrome扩展会**缓存**旧版本的HTML和JavaScript文件，简单的刷新无法清除缓存。

### 正确的重新加载方法：

#### 方法1：完全移除后重新加载（推荐）

1. **打开Chrome扩展程序页面**
   ```
   chrome://extensions/
   ```

2. **移除旧版本**
   - 找到"AI 圆桌 - Multi-AI Roundtable"
   - 点击**"移除"**按钮
   - 确认删除

3. **重新加载新版本**
   - 点击**"加载已解压的扩展程序"**
   - 选择文件夹：`~/cc/ai-roundtable-cn`
   - 点击"选择文件夹"

4. **验证加载成功**
   - 扩展列表中应该显示：**"AI 圆桌 中国版 - Multi-AI Roundtable CN"**
   - 版本号显示：**0.2.0**

#### 方法2：刷新按钮（可能不生效）

1. 在扩展程序页面找到"AI圆桌"卡片
2. 点击右下角的**刷新图标** 🔄
3. 检查版本号是否更新为0.2.0

⚠️ **如果版本号还是0.1.5，说明缓存未清除，请使用方法1**

## 🐛 故障排查

### 问题1：侧边栏还是显示旧版本UI

**原因**：HTML文件被缓存

**解决**：
1. 完全移除扩展（方法1）
2. 重新加载
3. 如果还是不行，重启Chrome浏览器

### 问题2：DeepSeek显示"Not found"

**原因**：DeepSeek页面未打开或content script未注入

**解决**：
1. 确保DeepSeek网站已打开并登录
2. 按F12打开控制台，查看是否有`[AI Panel]`日志
3. 刷新DeepSeek页面

### 问题3：消息无法自动发送

**原因**：输入框或发送按钮选择器不匹配

**解决**：
1. 按F12打开开发者工具
2. 查看控制台错误日志
3. 把错误信息发给我，我会调整选择器

### 问题4：无法捕获DeepSeek的回复

**原因**：响应容器选择器不匹配

**解决**：
1. 在DeepSeek发送一条测试消息
2. 查看HTML结构，找到回复内容的容器
3. 告诉我DOM结构，我会更新observer
