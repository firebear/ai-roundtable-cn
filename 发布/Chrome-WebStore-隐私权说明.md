# Chrome Web Store - 隐私权填写指南

## 一、单一用途说明 (Single purpose description)

**这是一个单独的输入字段，用于描述扩展的核心目的。**
要求：足够具体、易于理解、狭窄且单一。

```
让用户同时向多个 AI 平台发送问题，并在侧边栏集中查看所有 AI 的响应。
```

**要点**：
- ✓ 简洁明了（一句话）
- ✓ 描述**单一**核心目的：多平台发送 + 集中查看
- ✓ 易于理解
- ✓ 符合"狭窄且单一"的原则

**为什么不包含交叉评价功能？**
- 交叉评价是基于核心目的的**衍生功能**
- "单一用途"应该只描述**主要目的**，不是功能列表
- 核心目的就是：一键多发 + 集中显示

## 二、权限说明 (Permissions justification)

**每个权限都有单独的输入字段，需要分别说明理由。**

扩展共请求 3 个权限。

### 1. sidePanel（侧边栏权限）

**单一用途说明**:
```
提供侧边栏界面，让用户可以同时查看和管理多个 AI 平台的对话。
```

**详细说明**:
扩展使用 Chrome 侧边栏 API 创建一个持久的用户界面，用户可以在浏览其他网页的同时，方便地向多个 AI 平台发送消息并查看响应。侧边栏不会被页面内容覆盖，提供更好的多任务处理体验。

---

### 2. tabs（标签页权限）

**单一用途说明**:
```
查询标签页 URL 以检测哪些 AI 平台已被用户打开，并与 content scripts 通信。
```

**详细说明**:
扩展使用 tabs 权限来：
- 检测用户是否打开了支持的 AI 平台网站（共 7 个）
- 判断特定 AI 平台的连接状态（已连接/未连接）
- 在侧边栏中显示每个 AI 的在线状态
- 向 content scripts 发送消息（自动填充问题、捕获响应）

扩展**仅读取标签页的 URL**，不读取页面的内容、标题或浏览历史。

---

### 3. storage（存储权限）

**单一用途说明**:
```
在用户本地浏览器中存储 AI 响应内容，方便对比查看和会话管理。
```

**详细说明**:
扩展使用 chrome.storage.session API 在本地存储：
- 各个 AI 平台的最新响应内容
- 临时会话数据（仅在浏览器会话期间保留）

**重要声明**:
- 所有数据仅存储在用户本地浏览器中
- 不上传到任何远程服务器
- 用户关闭浏览器后，session 数据会自动清除
- 扩展不收集任何用户数据或使用行为信息

## 网站访问权限说明

### host_permissions（网站访问权限）

扩展请求访问以下 7 个 AI 平台网站：

| 网站 | 用途 |
|------|------|
| https://claude.ai/* | 在 Claude 网站上注入脚本，实现自动发送和响应捕获 |
| https://chatgpt.com/* | 在 ChatGPT 网站上注入脚本，实现自动发送和响应捕获 |
| https://chat.openai.com/* | 在 ChatGPT 旧版网站上注入脚本 |
| https://gemini.google.com/* | 在 Gemini 网站上注入脚本，实现自动发送和响应捕获 |
| https://chat.deepseek.com/* | 在 DeepSeek 网站上注入脚本，实现自动发送和响应捕获 |
| https://kimi.moonshot.cn/* | 在 Kimi 网站上注入脚本，实现自动发送和响应捕获 |
| https://www.kimi.com/* | 在 Kimi 新版网站上注入脚本 |
| https://chatglm.cn/* | 在 ChatGLM 网站上注入脚本，实现自动发送和响应捕获 |
| https://www.qianwen.com/* | 在通义千问网站上注入脚本，实现自动发送和响应捕获 |

**单一用途说明**:
```
仅在这些 AI 网站上注入必要的脚本，以实现自动发送消息和捕获响应功能。
扩展不会读取页面上的用户对话历史，仅在用户主动操作时访问当前页面。
```

**重要声明**:
- 扩展仅在用户**主动打开**这些网站时才会运行
- 扩展**不读取**页面的内容（除了捕获 AI 的新响应）
- 扩展**不修改**页面的任何内容（除了自动填充输入框）
- 扩展**不在后台**访问或监控这些网站

## 数据收集与使用声明

### 我们不收集的数据

- ❌ 个人身份信息（姓名、邮箱、电话等）
- ❌ 用户使用行为数据（点击、浏览历史等）
- ❌ 地理位置
- ❌ 设备信息
- ❌ IP 地址
- ❌ 用户与 AI 的对话内容（对话仅存储在本地）

### 我们处理的数据

| 数据类型 | 用途 | 存储位置 | 存储时长 |
|---------|------|---------|---------|
| 用户输入的问题 | 发送到 AI 平台 | 本地（临时） | 浏览器会话期间 |
| AI 的响应内容 | 展示给用户对比查看 | 本地（临时） | 浏览器会话期间 |
| AI 网站 URL | 检测连接状态 | 不存储 | 实时查询 |

**所有数据均存储在用户本地浏览器中，不会上传到任何服务器。**

## 隐私实践回答模板

### Chrome Web Store 可能会问的问题

**Q: 此扩展是否收集或使用用户数据？**
A: 否。此扩展不收集任何用户数据。所有数据仅存储在用户本地浏览器中。

**Q: 此扩展是否使用或传输用户数据以进行数字签名或认证？**
A: 否。

**Q: 此扩展是否与第三方共享用户数据？**
A: 否。此扩展不与任何第三方共享用户数据。

**Q: 此扩展是否允许用户在发送前控制数据？**
A: 是。用户在侧边栏输入问题后，可以查看并编辑内容，然后选择要发送到哪些 AI 平台。

**Q: 此扩展是否向用户展示其数据将如何使用？**
A: 是。扩展在 GitHub 仓库中提供了详细的隐私政策和使用文档。
